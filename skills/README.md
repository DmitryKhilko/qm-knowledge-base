# Структура skills

## Обзор
Описывается структура и содержимое раздела skills. В разделе приведены правила для AI-агентов, принимающих участие в 
создании требований, программного кода, автотестов для программного продукта.

## Структура директорий
```
skills/
├── spec/                                               # Каталог со skill для создания разного типа спецификаций
│   ├── new-feature/                                    # Каталог со skill для создания спецификации для новой Feature
│   │    ├── 01-business-problems-create-rules-ai.md    # Правила описания бизнес-проблемы
│   │    ├── 02-business-goals-create-rules-ai.md       # Правила описания бизнес-целей
│   │    ├── 03-business-kpi-create-rules-ai.md         # Правила описания критериев успеха
│   │    ├── 04-sub-features-create-rules-ai.md         # Правила выведения детальной функциональности из бизнес-требований
│   │    ├── 05-business-rules-create-rules-ai copy.md  # Правила выведения бизнес-правил 
... 
│   │    └── SKILL.md                                   # Правила (пошаговый алгоритм) создания спецификации
...
└── README.md                                           # Корневой README (текущий документ)
```

## Спецификация для новой основной функциональности системы
- **SKILL.md:** Правила (пошаговый алгоритм) создания спецификации для новой основной функциональности системы. 
- **01-business-problems-create-rules-ai.md:** Правила создания описания бизнес-проблемы в составе бизнес-требований.
- **02-business-goals-create-rules-ai.md:** Правила создания описания бизнес-целей в составе бизнес-требований. 
- **03-business-kpi-create-rules-ai.md:** Правила создания описания критериев успеха в составе бизнес-требований. 
- **04-sub-features-create-rules-ai.md:** Правила описания детальной функциональности (в составе основной 
функциональности) для группировки и типизации входящих в них требований (алгоритмы создания требований будут зависеть 
от типа детальной функциональности).
- **05-business-rules-create-rules-ai copy.md:** Правила создания полного набора BR; выделения общих BR (действующие 
на 2 и более FR/NFR/DR); преобразования оставшихся BR в FR/NFR/DR.
